## 1. 概述

网络地址用于标识网络上的各种资源。

在本教程中，我们将回顾 127.0.0.1 和本地主机之间的区别。首先，我们将定义它们是什么以及它们的使用地点和方式。最后，我们将描述这两个术语之间的区别。让我们从一些定义开始。

## 2.什么是127.0.0.1？

127.0.0.1是专门分配给loopback接口的IP地址，即loopback地址。127.0.0.1 是不可路由的私有 IP 地址，用于与输入该地址的同一设备建立连接。例如，![ping](https://www.baeldung.com/wp-content/ql-cache/quicklatex.com-ce8975f9d4e462a9e03daaec2deb4135_l3.svg) ![127.0.0.1](https://www.baeldung.com/wp-content/ql-cache/quicklatex.com-aef289dae5de678eac7676152c6a9707_l3.svg)在命令提示符窗口中输入命令将连接到始发设备：

```
C:用户>ping 127.0.0.1

使用 32 字节的数据 Ping 127.0.0.1：
来自 127.0.0.1 的回复：bytes=32 time<1ms TTL=128
来自 127.0.0.1 的回复：bytes=32 time<1ms TTL=128
来自 127.0.0.1 的回复：bytes=32 time<1ms TTL=128
来自 127.0.0.1 的回复：bytes=32 time<1ms TTL=128

127.0.0.1 的 Ping 统计数据：
数据包：发送 = 4，接收 = 4，丢失 = 0(0% 丢失)，
以毫秒为单位的近似往返时间：
最小值 = 0ms，最大值 = 0ms，平均值 = 0ms
```

那么为什么我们需要与始发设备建立连接呢？在某些情况下，我们想要验证连接到网络的设备上的互联网协议 (IP) 堆栈。它用于测试连接协议。此外，127.0.0.1 允许我们访问在原始机器上运行的网络服务。

现在，这在开发应用程序(尤其是 Web 应用程序)时特别有用。例如，开发人员可能托管一个应用程序 Web 服务器并将地址配置为 127.0.0.1。每次我们输入这个地址时，都会加载本地 Web 服务器的页面。

还必须注意，127.0.0.1 不是设备上存在的唯一环回地址。有一个与之相关的整个范围。因此，从 127.0.0.1 到 127.0.0.255 的任何地址都将以相同的方式运行。

## 3. 什么是本地主机？

术语[localhost](https://en.wikipedia.org/wiki/Localhost)通常用于指代环回地址为 127.0.0.1 的本地计算机。因此，在浏览器的地址栏中键入“localhost”将连接到设备本身。localhost 的用途是作为 127.0.0.1 的域名。

因此，我们可以说 localhost 具有与环回地址相同的功能。

## 4. 它们是如何工作的？

让我们来说明它们如何使用传输控制协议/Internet 协议 (TCP/IP) 模型。TCIP/IP 模型定义了 IP 网络中的通信标准。它由下图所示的四个主要层组成：

[![TCP层](https://www.baeldung.com/wp-content/uploads/sites/4/2021/12/tcpipimage.drawio-1.svg)](https://www.baeldung.com/wp-content/uploads/sites/4/2021/12/tcpipimage.drawio-1.svg)

OSI 模型中的 TCP 层

发送端的通信是自上而下的，接收端则相反。

发往发送方这两个地址中的任何一个的数据包将首先通过应用程序和传输层。到达网际层后，一旦检测到目的地址为127.0.0.1或localhost，则将数据包发送回设备。它不会退出本地网络。

## 5.有什么区别？

通常，localhost 和 127.0.0.1 可以互换使用。两者被认为是同一件事。但是，它们之间存在细微差别。让我们看看这些：

-   首先，必须解析或查找与 localhost 关联的 IP 地址，而 127.0.0.1 通常不需要解析。
-   其次，localhost 并不总是解析为 127.0.0.1。它可以解析为环回地址块内的另一个地址。
